📐 V4 Read Models (Views)

v_seo_opportunities
Aggregates seo_page_facts (30d). Classifies each page as: ranking_gap (pos 5–20), low_ctr (CTR < 2%, impr > 100), high_rank_low_traffic, or monitor. Sorted by opportunity_score DESC = impressions × (1−CTR).
v_seo_cannibalization
Finds queries answered by 2+ pages. Filters: seo_query_facts 30d, impr > 50, COUNT(DISTINCT page_url) ≥ 2. Returns query, competing pages array, best position.
v_seo_technical_health
Aggregates seo_issues per brand/website. Computes health_score = 100 − (critical×20 + high×10 + medium×3 + low×1). Buckets by issue_type.
v_seo_business_impact
JOINs seo_page_facts + analytics_page_facts on brand_id + page_path. Returns revenue_30d, conversions, revenue_per_session, bounce_rate per URL. The SEO → Revenue connector.
v_seo_brief_pipeline
JOINs seo_content_briefs + seo_keyword_clusters + seo_pages. Ordered by priority (critical first), then due_date ASC. Powers the content brief Kanban/pipeline view.
🔧 V4 Auto-Detection Functions
detect_low_ctr_issues(brand_id, connection_id) — Auto-inserts on-page/low-ctr issues from v_seo_opportunities. Uses ON CONFLICT DO NOTHING for idempotency.
refresh_keyword_opportunity_scores(brand_id) — UPDATEs seo_keyword_map with latest position/impressions/CTR/opportunity_score from seo_query_facts (30d aggregate).
